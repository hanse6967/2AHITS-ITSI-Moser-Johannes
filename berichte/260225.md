# Übung 3.2: Textviewer (more & less)

## Befehle und Steuerung
Öffnen: more /pfad/zur/datei oder less /pfad/zur/datei.

Bewegen: * Leertaste: Eine Seite vorwärts.

Enter: Eine Zeile vorwärts.

b (nur bei less): Eine Seite zurück.

Suchen: Mit der Taste /, gefolgt vom Suchbegriff, kann man im Text suchen. Mit n springt man zum nächsten Treffer.

Beenden: Die Taste q schließt das Tool.

Unterschied zwischen more und less
Obwohl der Name "less" (weniger) vermuten lässt, kann das Tool tatsächlich mehr als "more".

more: Liest die Datei sequenziell. Man kann fast nur vorwärts blättern.

less: Lädt nicht die gesamte Datei auf einmal in den Speicher (schneller bei riesigen Dateien) und erlaubt das freie Scrollen nach oben und unten.

Merksatz: "Less is more, but more than more."

# Übung 3.3: Wildcards (Platzhalter)
Wildcards sind Symbole, die vom System als Platzhalter für andere Zeichen interpretiert werden.

Der Stern *: Er steht für eine beliebige Anzahl an Zeichen (auch null).

### Anwendung:

ls *.md: Listet alle Dateien auf, die auf .md enden.

ls *.txt: Listet alle Dateien auf, die auf .txt enden.

ls Übung*: Findet alles, was mit "Übung" beginnt.

# Übung 3.4: Der find-Befehl

Das Tool find ist extrem mächtig, um das Dateisystem nach bestimmten Kriterien zu durchsuchen.

Praktische Suchabfragen (vom Home-Verzeichnis ~ aus)

1. Verzeichnis "profiles" suchen: find ~ -type d -name "profiles"

2. Datei ".latest.json" suchen: find ~ -type f -name ".latest.json"

3. Alle JSON-Dateien: find ~ -type f -name "*.json"

4. Nur Verzeichnisse: find ~ -type d

5. Markdown-Files (letzte 4 Wochen geändert): find ~ -type f -name "*.md" -mtime -28

6. Verzeichnisse (letzte 4 Wochen geändert): find ~ -type d -mtime -28

7. Verzeichnisse (älter als 4 Wochen): find ~ -type d -mtime +28

8. Dateien mit Berechtigung 644: find ~ -type f -perm 644
   Bedeutung 644: Der Besitzer darf lesen/schreiben (6), die Gruppe nur lesen (4) und der Rest der Welt ebenfalls nur lesen (4).

9. Große Dateien in /usr (> 2MB): find /usr -type f -size +2M

## Erweiterte Aktionen mit -exec
In ~/workspace wurden Dateien erstellt. Um alle .txt-Dateien zu löschen, die kleiner als 10 Bytes sind:
find . -name "*.txt" -size -10c -exec rm {} \;
(Hierbei ist {} der Platzhalter für die gefundene Datei und \; schließt den Befehl ab.)

## Zusammenfügen von Markdown-Dateien:
Um alle .md-Inhalte in eine neue Datei komplett.md zu schreiben:
find . -name "*.md" -exec cat {} + > komplett.md

### Detaillierte Liste von Dateien in /var mit "log" im Namen:
find /var -type f -name "*log*" -exec ls -l {} +

# Übung 3.5: Das Tool "file"
Das Kommando file bestimmt den Dateityp, unabhängig von der Dateiendung (da Linux-Endungen oft nur zur Orientierung für den User dienen).

Analyse der Datei "qwert"
Wenn eine Datei namens qwert keine Endung hat, hilft der Befehl:
file qwert

Das Tool analysiert die "Magic Numbers" (die ersten Bytes) der Datei. Wenn file meldet, dass es sich um "ASCII text" handelt, kann man den Inhalt einfach mit cat qwert oder less qwert lesen. Falls es eine komprimierte Datei ist (z.B. "gzip compressed"), müsste man sie erst mit gunzip entpacken, um an den Text zu gelangen.

Ein paar der wichtigsten Schalter

-name  :    Suche nach dem Dateinamen (mit Wildcards)

-type f: 	Suche nur nach Dateien (files)

-type d: 	Suche nur nach Verzeichnissen (directories)
