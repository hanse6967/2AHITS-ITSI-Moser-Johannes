# 1. Schritt: Deine Arbeit hochladen

Wenn du an deinem Bericht etwas geändert hast, musst du das "speichern" und ins Internet schicken:

Git öffnen: Klicke in Replit auf das + und such nach Git.

Notiz schreiben: Schreib bei "Summary" kurz rein, was du gemacht hast (z. B. "Bericht fertig").

Speichern: Klick auf den blauen Button Stage and commit all changes.

Abschicken: Klick oben auf Sync Changes. Jetzt fliegen die Daten zu GitHub.

Checken: Wenn du bei GitHub unter Actions einen grünen Haken siehst, ist dein Bericht online.

# 2. Schritt: Deinen Haupt-Link finden

Du musst wissen, wie die Adresse deiner Webseite überhaupt heißt:

Geh auf GitHub zu deinem Projekt und klick oben auf Settings (das Zahnrad).

Klick links in der Liste auf Pages.

Dort siehst du einen Link unter "Your site is live at...". Das ist die Adresse deiner Seite. Kopier sie dir.

# 3. Schritt: Den Link zum Bericht basteln

Damit man genau deinen Bericht sieht und nicht nur die Startseite, musst du den Link anpassen:

Pfad merken: Schau bei GitHub, in welchem Ordner dein Bericht liegt (z. B. im Ordner Berichte) und wie die Datei heißt (z. B. mein-tag.md).

Zusammenfügen: Häng diesen Namen einfach hinten an deinen Haupt-Link dran.

Wichtig – Das Ende ändern: Dein Computer speichert die Datei als .md, aber im Internet muss sie auf .html enden, damit sie wie eine echte Webseite aussieht.

Beispiel: Aus .../Berichte/test.md wird im Browser .../Berichte/test.html.
