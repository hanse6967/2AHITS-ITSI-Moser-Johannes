### Analyse mit dem GNU Debugger

In diesem Abschnitt wird das Programm untersucht, um die Passwortprüfung zu umgehen.
Dazu wird **Reverse Engineering** verwendet, also das Analysieren eines Programms, um dessen Funktionsweise zu verstehen.

Nachdem die Struktur verstanden wurde, wird das Programm im Speicher verändert, bevor es vollständig ausgeführt wird.

Der **GNU Debugger (gdb)** ermöglicht es, Programme Schritt für Schritt auszuführen und ihr Verhalten zu beobachten.

---

### Wichtige gdb-Befehle

* `b main` → Haltepunkt bei der main-Funktion setzen
* `run` → Programm starten
* `disas main` → Assemblercode der main-Funktion anzeigen
* `si` → einzelne Maschinenbefehle ausführen
* `info registers` → Registerwerte anzeigen

Damit kann man den Ablauf des Programms genau verfolgen.

---

### Untersuchung der Passwortprüfung

Mit `disas main` wird sichtbar, welche Befehle tatsächlich ausgeführt werden.

Eine entscheidende Stelle ist:

```
cmp $0x0,%eax
je 1263
```

Hier wird ein Wert verglichen. Ist das Ergebnis 0, wird ein Sprung ausgeführt → das Passwort gilt als falsch.

---

### Veränderung der Bedingung

Die Passwortabfrage kann im Speicher manipuliert werden:

```
set {unsigned char}0x11c8 = 0x90
```

Der Wert `0x90` steht für einen sogenannten **NOP-Befehl** (keine Operation).
Dadurch wird der Sprungbefehl überschrieben und nicht mehr ausgeführt → jedes Passwort wird akzeptiert.

---

### Übung: Addition zu Subtraktion ändern

Nach der Passwortprüfung verarbeitet das Programm zwei Zahlen und addiert sie.
Diese Funktion kann so verändert werden, dass stattdessen eine Subtraktion erfolgt.

Dazu wird im Assemblercode die entsprechende Stelle gesucht und der Additionsbefehl durch einen Subtraktionsbefehl ersetzt.
Ergebnis: Statt `a + b` wird nun `a - b` berechnet.

---

### Dauerhafte Änderung im Programm (Binary Patching)

Die Änderungen lassen sich auch direkt in der Datei speichern.
Dazu wird mit `objdump` der Code analysiert, um die richtige Position zu finden.

Beispiel:

```
11c8: 0f 84 95 00 00 00
```

Dieser Befehl ist ein bedingter Sprung (**je = jump if equal**).

---

### Patchen mit Hexeditor

Mit einem Hexeditor kann diese Stelle direkt verändert werden:

1. Datei öffnen:

   ```
   hexedit pwcheck
   ```
2. Mit `Ctrl + G` zur Adresse `11c8` springen
3. Bytes ersetzen:

   ```
   0f 84 95 00 00 00
   →
   90 90 90 90 90 90
   ```

Da `0x90` für NOP steht, wird der Sprung entfernt.
Folge: Die Passwortprüfung wird komplett übersprungen.
